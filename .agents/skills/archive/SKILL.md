# 待归档内容归档

## 工作任务

你需要把待归档的内容按照项目的文档管理规则归档。文档重视玩法设计和技术实现，轻视修改记录，因此并不是所有内容都必须归档。

宿主项目的文档管理规则通常位于 `.harness/doc/`（如 `doc-manager-rule.md`）。

## 归档原则

- 只记录重要决策，不记录实现细节
- 避免拖沓的描述，保持内容精炼
- 专注于架构层面的长期价值

## 退出条件

- 待归档内容已写入规则允许的目录（模块 `docs/` 或根 `docs/`），符合放置速查。
- 已更新 `docs/index.md`（若新增或移动了文档）。
- 明确说明哪些内容**未**归档及理由（实现细节、临时讨论等）。

## 反合理化

| 借口 | 反驳 | 级别 |
|------|------|------|
| "聊天里说过就等于归档了" | 未落盘到项目文档则对后续 agent 不可见。 | 重要 |
| "全部记下来以后好查" | 只记录重要决策；实现细节与修改流水不归档。 | 重要 |
| "随便放个 md 就行" | 须遵守文档管理规则的目录与命名，并更新索引。 | 重要 |
| "这点小事不值得归档" | 若影响架构或长期约定，精炼一段即可；若不影响，在退出条件中声明未归档。 | 重要 |
