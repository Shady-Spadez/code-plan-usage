# 整理文档

## 工作任务

整理仓库内 `.md` 文档（根 `docs/`、各模块 `docs/` 与 `README.md`）。

## 工作步骤

1. 随机抽查一个 `.md` 文档。
2. 将抽查的文档与代码库对照比对，判断该文档是否已过期。
   - 如果文档过期：
     a. 若可以更新，则及时更新，使其与代码保持同步；
     b. 若已完全没有对应代码，则删除此文档。
3. 整理文档，包括但不限于：
   - 删除冗余的内容或文件
   - 删除项目文档管理规则不允许记录的内容或文件
   - 合理地合并零碎的内容
   - 拆分过长的文件
   - 重命名或移动文件
   - 重新组织目录结构
   - 清理实现细节和过程记录，保留关键决策
4. 更新 `docs/index.md` 的索引内容

工作的要求参考项目的文档管理规则（通常位于 `.harness/doc/`，如 `doc-manager-rule.md`）。

## 退出条件

- 至少完成一轮「抽查 → 对照代码 → 更新或删除」。
- 若重组了目录或移动了文件，已更新 `docs/index.md`。
- 删除文档前已确认无对应代码或内容已迁移。

## 反合理化

| 借口 | 反驳 | 级别 |
|------|------|------|
| "文档看起来旧，但不敢删" | 对照代码：无对应实现则删；有实现则更新，不搁置。 | 重要 |
| "索引以后再说" | 移动/重命名后必须同步 `docs/index.md`，否则索引即谎言。 | 重要 |
| "多抽查几篇太费时" | 每轮至少一篇；organize 是持续整理，不是一次清完仓库。 | 重要 |
| "过程记录也有价值" | 文档管理规则要求清理实现细节与过程记录，保留关键决策。 | 重要 |
